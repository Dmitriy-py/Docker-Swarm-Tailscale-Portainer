### Docker-Swarm как основа

### Portainer — это легковесный инструмент управления контейнерами, предоставляющий централизованную веб-панель для развертывания, мониторинга и управления контейнерами, томами, сетями и кластерами (включая Docker Swarm и Kubernetes). Он значительно упрощает администрирование и визуализирует состояние сложных распределенных систем.

## 1. Проект “Распределенная ферма вычислений”

### Мы можем использовать свой кластер для задач, требующих много процессорного времени, которые наш ПК в одиночку тянет долго:
```
Рендеринг видео или 3D: Запускаем контейнеры с Blender или FFmpeg. Разделите исходный файл на части (chunks), и пусть ваш кластер обрабатывает их параллельно.
Сбор данных (Парсинг/Scraping): Если нам нужно собирать данные с 1000 сайтов, запустите 10 контейнеров-парсеров. Мастер будет балансировать нагрузку между всеми узлами.
```
## 2. Проект “Личное защищенное облако” (Self-Hosted)

### Перестанем платить за Google Drive или Dropbox:
```
Nextcloud: Разверните в кластере (через Stacks/Docker Compose) Nextcloud. Данные будут лежать на сервере в облаке, но мы можем подключить локальные диски (через NFS) как хранилище.
Bitwarden (Vaultwarden): Свой собственный менеджер паролей. Никто, кроме нас, не имеет к нему доступа.
```

### 3. Проект “Мониторинг всего на свете”
```
Стек Prometheus + Grafana + Loki: “Сколько процентов CPU использует vm-2 сейчас”, “Сколько памяти ест ваш сервис Nginx”.
```

### 4. Проект “Собственный VPN-сервис или Прокси”
```
Так как у нас есть Tailscale, мы можем внутри кластера поднять свои прокси-сервера (например, Shadowsocks или V2Ray), чтобы иметь “чистый” выход в интернет с IP-адресом вашего облачного сервера из любой точки мира.
```

### 5. Проект “CI/CD Конвейер”

#### Мы можем сделать так, чтобы наш кластер сам собирал наш код и выкладывал его в интернет:
```
Установим Gitea (свой GitHub) + Drone CI или Jenkins.
Суть: Мы пишем код на Windows -> делаем git push -> наш кластер в облаке подхватывает код -> собирает Docker-образ -> развертывает его на vm-2.
```
